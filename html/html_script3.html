Javascript uses a total of 7 Primitive and 1 Non-Primtive data types. The Primitive data types include:

  • Number
  • String
  • Boolean: true and false vlaues: Everythign with a content is true and everything empty is false
  • Null: used to represent intentional absence of any object value.

object: Collections of key-value pairs or complex data. Another subtype commonly used is Array. The string, number and Boolean datatype can be created using the new constructor (String, Number, Boolean). Also we can use the BigInt() constructor with a string to declare a bigint.

Functions in JS can be defined using either of the three ways and all three behave differently in terms of thier use-cases.

  1. Function definiton

    Hoisted(conceptually moving to the top of where they are declared in) to the top of their scope, meaning they can be called before they are defined in the code.

                    function fizzBuzz(){
                    let output= []; 
                    for(let i =1 ; i<=100; i++){
                        if(i%3===0 && i%5===0){output.push("FizzBuzz");}
                        else if(i%5===0){output.push("Buzz");}
                        else if(i%3===0){output.push("Fizz");}
                        else{output.push(i);}  
                    }
                    console.log(output)
                }

                fizzBuzz();

  2. Fucntion Expression:

    Assigned to a variable. They are not hoisted in the same way as declarations (they follow the hoisting rules of the variable declaration like let, const, or var).

                    const add = function(a,b,c){return a+b+c;}
                const calcAge = function(birthYear){
                    const currentYear = new Date().getFullYear(); 
                    return currentYear - birthYear; 
                }

                let z= add(2,3,4);
                console.log(z); 
                let age = calcAge(2006); 
                console.log(age);

  3. Arrow Fucntions

    A shorter syntax for writing functions. Crucially, they do not have their own this context (they inherit this from the enclosing lexical scope) and do not have an arguments object.

                    const multiply = (a, b) => a * b;
                multiply(3,4);

  4. Anonymous Fucntions

    Functions without a name. Often used as callbacks or inside IIFEs.

                    setTimeout(function() {
                    return ("Executed after 1 second");
                }, 1000);

  5. IIFEs(ImmediateLY Invoked Function Expression)

    Functions that run as soon as they are defined. Used to create a private scope and avoid polluting the global namespace.

                    (function() {
                    console.log("IIFE executed!");
                })();

  6. Generator Function

    Functions that can be paused and resumed using the function* syntax and the yield keyword. They return an Iterator object.

                    function* countUp() {
                    yield 1;
                    yield 2;
                    yield 3;
                }
