There are four broad categories which we look at when we're determining the overall level of importance of a CSS rule.

1. Position of the CSS rule

for example : 
        li{
            color: red;
            color: green;
        }
        li{
            color: blue;
        }
The blue color will be applied to the list element, as it is the last one appplied. The lower down in the CSS external file or in an internal style element, the more important it is.

2. Specificty of the rule

Say for example for a list element of html has the following list element:
  • This is a list element
    •         li{ #Element specificity
            color: blue; 
        }
        .first-class{ #class specificity
            color: red;
        }
        li[draggable]{ #attribute specificity
            color: purple
        }
        #first-id{ #id specificity
            color: orange
        }
    Thus we see only the color orange is applied as an id assigned to an element has the highest level of specificity as CSS rules allow for only one unique id. Order of specificity thus is :

    element< class< attribute< id

    The inline> internal > external flow of the relevenace of type of CSS rule applied

    4. Importance of CSS rule<: !important keyword
