Font Properties

Font Sizes

Font sizes can be given in multiple types for example:

  • Pixels: px => 1/96th of an inch (20 px size)
  • Points: pt => 1/72nd of an inch (20 pt size)
  • 100% of parent: em => it means the relative percentage factor of the body elements's size. (1 em size)
  • 100% of root: rem => relative to the root of the html file (2 rem size)

Font Weight

Sets the thickness of the text chracter. Font weight can be used in three different types :

  • For example: Normal/bold (Bold weight applied)
  • Relative to parent: for example: lighter/ bolder (lighter font weight applied). Lighter reduces the font weight number by 100 and bolder increases it by 100.
  • Number : for example: 100 to 900 in th range of light to bold (Font weight of 700 applied)

Font Family

Determines what your text looks like. it is specified in css by seperating A typeface that accepts one or more font names, separated by commas. The browser attempts to use the first font in the list that is available on the user's computer or can be downloaded.
The list awlays end with a generic font family as a fallback option if any of the specific fonts are not available.
The font typeface should be written in qoutes if it has more than one word. example : "Times New Serif" , sans-serif
The different font families ( that are used as generic fallback fonts are as follows):

  • serif: Fonts with small strokes (serifs) at the edges of each letter
  • sans-serif: Fonts with clean lines and no strokes.edges at right angle.
  • monospace: Fonts where all letters have the same fixed width, often used for code. (Typewriter font)
  • cursive: Fonts that imitate human handwriting
  • fantasy: Decorative or playful fonts

Text Align

Another css property that allows the text to be navigated along a line either on the right, left, center or we can use :

  • justify: Stretches the lines of text so that each line has equal width, aligning text with both the left and right margins
  • start: Aligns text to the start edge of the element (left in a left-to-right context, right in a right-to-left context).
  • end: Aligns text to the end edge of the element (right in a left-to-right context, left in a right-to-left context).
